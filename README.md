# svg2htmlsvg

Convert standalone SVG files to inline HTML-embeddable `<svg>` elements.

## Overview

`svg2htmlsvg` is a robust, production-quality zsh script that transforms standalone SVG files (with XML declarations, DOCTYPEs, and other preamble) into clean, inline-ready `<svg>` elements suitable for embedding directly in HTML documents.

### Why?

Most SVG editors and export tools produce standalone SVG files with XML declarations, DOCTYPE tags, and other markup that prevents clean inline embedding.  Manually cleaning these up is tedious and error-prone — especially when you also want to strip comments, remove metadata, inject CSS classes, or minify the output.  This tool automates the entire pipeline.

### Key Features

- **Strips XML declarations** (`<?xml ... ?>`) by default
- **Strips DOCTYPE declarations** (`<!DOCTYPE ...>`) by default
- **Strips comments** (`<!-- ... -->`) on demand, including multi-line
- **Strips `<metadata>` elements** on demand, including multi-line
- **Injects attributes** — add `class` and/or `id` to the root `<svg>` element
- **Minifies** — collapses whitespace for production embedding
- **Wraps in HTML5** — optionally outputs a complete HTML5 document
- **Dry-run mode** — preview the transformation pipeline without writing output
- **Multi-level verbosity** — from silent operation to full trace logging
- **Cross-platform** — macOS, Linux, and Windows (via WSL2/MSYS2)

Built on the [Shell-Script-Templates](https://github.com/soren42/Shell-Script-Templates) framework (v3.0).

## Quick Start

```bash
# Clone the repo
git clone https://github.com/soren42/svg2htmlsvg.git
cd svg2htmlsvg

# Make it executable
chmod +x svg2htmlsvg.zsh

# Basic conversion (output to stdout)
./svg2htmlsvg.zsh logo.svg

# Strip comments, minify, write to file
./svg2htmlsvg.zsh --strip-comments --minify -o logo-inline.html logo.svg

# Wrap in a full HTML5 page with a title and CSS class
./svg2htmlsvg.zsh --wrap-html --title "Company Logo" --class "logo-main" logo.svg

# Preview what would happen (dry-run with verbose)
./svg2htmlsvg.zsh -vv --dry-run logo.svg
```

### Install Globally (Optional)

```bash
# Symlink into your PATH
ln -s "$(pwd)/svg2htmlsvg.zsh" /usr/local/bin/svg2htmlsvg

# Or copy directly
cp svg2htmlsvg.zsh /usr/local/bin/svg2htmlsvg
chmod +x /usr/local/bin/svg2htmlsvg

# Optionally compile for faster loading
zcompile svg2htmlsvg.zsh
```

## Usage

```
svg2htmlsvg [OPTIONS] <input.svg>
```

### Standard Options

|Short|Long           |Description                              |
|-----|---------------|-----------------------------------------|
|`-h` |`--help`       |Show help message and exit               |
|`-V` |`--version`    |Show version information and exit        |
|`-v` |`--verbose`    |Increase verbosity (repeatable: `-vvv`)  |
|`-q` |`--quiet`      |Suppress all non-error output            |
|`-n` |`--dry-run`    |Show what would be done without executing|
|`-d` |`--debug`      |Maximum verbosity plus shell xtrace      |
|`-c` |`--config FILE`|Use specified configuration file         |
|`-o` |`--output FILE`|Write output to FILE instead of stdout   |

### SVG Transformation Options

|Long              |Description                                                    |
|------------------|---------------------------------------------------------------|
|`--strip-comments`|Remove all XML/SVG comments                                    |
|`--strip-metadata`|Remove `<metadata>...</metadata>` elements                     |
|`--minify`        |Collapse unnecessary whitespace                                |
|`--wrap-html`     |Wrap output in a minimal HTML5 document                        |
|`--class CLASS`   |Add a CSS class to the root `<svg>` element                    |
|`--id ID`         |Add an `id` attribute to the root `<svg>` element              |
|`--title TITLE`   |Set `<title>` in the HTML wrapper (implies `--wrap-html`)      |
|`--keep-xmldecl`  |Preserve the `<?xml ...?>` declaration (stripped by default)   |
|`--keep-doctype`  |Preserve the `<!DOCTYPE ...>` declaration (stripped by default)|

## Examples

### Basic Inline Conversion

**Input** (`icon.svg`):

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE svg PUBLIC "-//W3C//DTD SVG 1.1//EN" "http://www.w3.org/Graphics/SVG/1.1/DTD/svg11.dtd">
<!-- Generated by Inkscape -->
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24">
  <metadata>
    <rdf:RDF>...</rdf:RDF>
  </metadata>
  <circle cx="12" cy="12" r="10" fill="#333"/>
</svg>
```

**Command:**

```bash
./svg2htmlsvg.zsh icon.svg
```

**Output:**

```html
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24">
  <metadata>
    <rdf:RDF>...</rdf:RDF>
  </metadata>
  <circle cx="12" cy="12" r="10" fill="#333"/>
</svg>
```

### Full Cleanup with Class Injection

```bash
./svg2htmlsvg.zsh --strip-comments --strip-metadata --class "icon icon-lg" --id "main-icon" icon.svg
```

**Output:**

```html
<svg id="main-icon" class="icon icon-lg" xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24">
  <circle cx="12" cy="12" r="10" fill="#333"/>
</svg>
```

### Production Minification

```bash
./svg2htmlsvg.zsh --strip-comments --strip-metadata --minify -o icon.min.svg icon.svg
```

### Wrapped HTML5 Document

```bash
./svg2htmlsvg.zsh --wrap-html --title "Dashboard Icon" --class "widget-icon" icon.svg
```

**Output:**

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Dashboard Icon</title>
</head>
<body>
<svg class="widget-icon" xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24">
  <circle cx="12" cy="12" r="10" fill="#333"/>
</svg>
</body>
</html>
```

## Configuration Files

`svg2htmlsvg` supports configuration files loaded from standard locations in priority order:

1. `/etc/svg2htmlsvg/svg2htmlsvg.conf`
1. `/etc/svg2htmlsvg.conf`
1. `~/.config/svg2htmlsvg/svg2htmlsvg.conf`
1. `~/.svg2htmlsvg.conf`
1. `./svg2htmlsvg.conf`
1. `$SVG2HTMLSVG_CONFIG_FILE` (environment variable)
1. `--config FILE` (command-line argument — highest priority)

Configuration files are shell-sourceable.  Example:

```bash
# ~/.config/svg2htmlsvg/svg2htmlsvg.conf
OPT_STRIP_COMMENTS=true
OPT_STRIP_METADATA=true
OPT_MINIFY=true
```

### Dependencies

**Required:**

- `zsh` 5.0 or later
- `sed` (GNU sed preferred; BSD sed on macOS works)
- `awk` (gawk, mawk, or POSIX awk)

**Optional:**

- `xmllint` — enables well-formedness validation of input SVGs

## Architecture

The script uses a streaming pipeline architecture: input flows through a series of transformation stages, each implemented as a function reading from stdin and writing to stdout.  Stages that are disabled become identity passes (`cat`).

```
input.svg
  → [Strip XML declaration]
  → [Strip DOCTYPE]
  → [Strip comments]
  → [Strip metadata]
  → [Inject class/id]
  → [Minify]
  → [Wrap in HTML5]
  → output (stdout or file)
```

This design is deliberately simple and composable.  Each stage can be tested independently, and the pipeline can be extended by adding new stages.
